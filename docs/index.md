# Terraform Registry Provider

The **registry** provider manages resources within a self-hosted [Terraform Registry Backend](https://github.com/terraform-registry/terraform-registry-backend) instance. It allows you to declaratively manage all registry resources via Terraform.

## Features

- **Identity & Access**: Users, organizations, organization members, API keys, role templates
- **Registry Content**: Module records, provider records, SCM integrations, module-SCM links
- **Infrastructure**: Provider mirrors, Terraform binary mirrors, storage configurations
- **Governance**: Policies, approval requests
- **Data Sources**: Read and list any of the above resources, plus audit logs and statistics

## Example Usage

```terraform
terraform {
  required_providers {
    registry = {
      source  = "registry.terraform.io/terraform-registry/registry"
      version = "~> 0.1"
    }
  }
}

provider "registry" {
  endpoint = "https://registry.example.com"
  token    = var.registry_token
}

variable "registry_token" {
  description = "API key or JWT token for the registry."
  type        = string
  sensitive   = true
}
```

## Schema

### Optional

- `endpoint` (String) The base URL of the Terraform Registry Backend (e.g., https://registry.example.com). Can also be set with the TF_REGISTRY_ENDPOINT environment variable.
- `insecure` (Boolean) Disable TLS certificate verification. Should only be used in development environments.
- `max_retries` (Number) Maximum number of retries for failed requests (429, 5xx). Defaults to 3.
- `timeout` (Number) HTTP request timeout in seconds. Defaults to 30.
- `token` (String, Sensitive) API key or JWT bearer token for authentication. Can also be set with the TF_REGISTRY_TOKEN environment variable.

## Authentication

The provider supports two authentication methods, both passed as a Bearer token:

1. **API Key** — Created via the registry admin UI or API. Recommended for CI/CD and automation.
2. **JWT** — Issued after OIDC login. Suitable for interactive use.

Set the token via the `token` attribute or the `TF_REGISTRY_TOKEN` environment variable.
